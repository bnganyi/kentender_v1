# KenTender — Budget Module PRD v1

**1\. Purpose**

The Budget module defines and manages **planned financial allocations** aligned to a Strategic Plan.

It enables:

- creation of annual budgets
- allocation of funds to Strategy Programs
- visibility of total vs allocated amounts
- preparation for downstream procurement planning

**2\. Scope (strict)**

**Included in v1**

- Budget creation
- Budget listing (workspace)
- Budget selection (master-detail)
- Budget builder (Program-level allocation)
- Allocation persistence
- Totals calculation (allocated vs remaining)
- Role-based visibility (reuse Strategy roles)

**Excluded from v1**

- Objective-level allocation
- budget approval workflows
- budget revisions/versioning
- commitment tracking
- procurement integration
- multi-year budget modeling
- financial accounting / GL integration

**3\. Core concepts**

**3.1 Budget**

A Budget represents:

A financial allocation plan for a specific fiscal year, aligned to a Strategic Plan.

**3.2 Budget Allocation**

A Budget Allocation represents:

A portion of the total budget assigned to a specific Strategy Program.

**4\. User roles involved**

Reuse existing roles.

| **Role** | **Access** |
| --- | --- |
| Strategy Manager | Full create/edit |
| Planning Authority | Read-only |
| Requisitioner | No access |
| Procurement Planner | No access (v1) |
| Finance Reviewer | No access (v1) |
| Administrator | Full |

**5\. User flows (authoritative)**

**Flow 1 — Create Budget**

**Actor**

Strategy Manager

**Steps**

1.  Open Budget workspace
2.  Click “New Budget”
3.  Fill:
    - Budget Name
    - Fiscal Year
    - Strategic Plan
    - Currency
    - (optional) Total Budget Amount
4.  Save

**Result**

- Budget appears in workspace list
- Budget is selectable
- Status = Draft

**Flow 2 — Allocate Budget**

**Actor**

Strategy Manager

**Steps**

1.  Select Budget from workspace
2.  Click “Open Budget Builder”
3.  See Programs from linked Strategic Plan
4.  Enter allocation amount per Program
5.  Save

**Result**

- Allocations persist
- Totals update

**Flow 3 — Review Budget**

**Actor**

Strategy Manager / Planning Authority

**Steps**

1.  Select Budget
2.  View:
    - total budget
    - allocated amount
    - remaining amount
    - allocations per program

**Result**

- user understands distribution and completeness

**9\. Validation rules (authoritative)**

**9.1 Budget validation**

**Rule BUD-1**

budget_name required

**Rule BUD-2**

fiscal_year must be valid integer

**Rule BUD-3**

strategic_plan must exist

**9.2 Allocation validation**

**Rule ALLOC-1**

amount > 0

**Rule ALLOC-2**

program.strategic_plan == budget.strategic_plan

**Rule ALLOC-3**

Uniqueness:

(budget, program) must be unique

**Rule ALLOC-4 (if total defined)**

If total_budget_amount exists:

sum(allocation.amount) ≤ total_budget_amount

**10\. Computed values**

These are NOT stored, but computed.

**Allocated Amount**

sum(all allocations for budget)

**Remaining Amount**

if total_budget_amount exists:

total_budget_amount - allocated_amount

else:

null
