# Budget Module — Scope Summary (MVP v1)

**Core purpose**

Translate Strategy into **allocations and financial intent**.

Not execution. Not procurement. Not accounting.

Just:

“Given this strategy, how much are we planning to spend, where, and when?”

**1\. What Budget is (and is not)**

**Budget IS**

- Structured financial planning
- Linked to Strategy hierarchy
- Time-bound (fiscal years)
- Versioned (Draft → Approved later)
- Aggregated (Program / Objective level)

**Budget is NOT (v1)**

- General ledger
- Payment tracking
- Procurement execution
- Commitment accounting
- Forecasting engine

Keep it clean.

**2\. Core user flows (MVP)**

There are only **3 flows that matter**.

**Flow 1 — Create Budget**

- User creates a Budget (e.g. “FY2026 Budget”)
- Defines:
    - fiscal year
    - linked Strategic Plan

**Flow 2 — Allocate Budget**

- User opens Budget Builder
- Allocates amounts to:
    - Programs
    - (optionally Objectives in v1.1)

**Flow 3 — Review Budget**

- See totals:
    - by Program
    - overall
- Validate against Strategy scope

That’s it.

Anything beyond this is scope creep right now.

**4\. Minimal domain model**

Keep this extremely tight.

**A. Budget**

|     |     |     |     |
| --- | --- | --- | --- |
| **Field** | **Type** | **Req** | **Notes** |

|     |     |     |     |
| --- | --- | --- | --- |
| **Field** | **Type** | **Req** | **Notes** |

| order_index | Int | ✓   |     |

**Important constraints**

- Allocation must belong to **same Strategic Plan**
- One allocation per program per budget (v1)
- No Objective-level allocation yet (optional later)

**5\. Key validation rules**

**Budget level**

- fiscal_year must be valid (e.g. ≥ current year - 1)
- strategic_plan must exist

**Allocation level**

**Rule B1**

allocation.program.strategic_plan == budget.strategic_plan

**Rule B2**

amount > 0

**Rule B3 (optional v1)**

If total_budget_amount exists:

- sum(all allocations) ≤ total_budget_amount

**Rule B4**

No duplicate:

(budget, program) must be unique

**9\. Critical design decisions (do NOT skip)**

**1\. No Objective-level allocations (yet)**

This is intentional.

Otherwise:

- complexity explodes
- UX breaks
- validation becomes messy

Add later only if needed.

**2\. No versioning (yet)**

Only:

- Draft

Later:

- Approved
- Revised

**3\. No workflow (yet)**

No approvals, no submission states.

**4\. No integration with Procurement yet**

Budget exists independently.

Later:

- procurement references budget
- commitments reduce available budget

**10\. Acceptance criteria (exit gate)**

You do not leave Budget MVP until:

1\. Budget workspace loads

2\. Budgets can be created

3\. Budgets appear in list

4\. Budget selection updates detail panel

5\. Builder opens from selected budget

6\. Program allocations can be created

7\. Allocations persist correctly

8\. Totals calculate correctly

9\. Strategy Manager can edit

10\. Planning Authority cannot edit

11\. Requisitioner cannot see Budget

12\. Smoke tests pass

**Bottom line**

Strategy = WHAT

Budget = HOW MUCH

Procurement = HOW

You’re now entering the **financial control layer**.

If you keep Budget simple and disciplined, everything downstream stays sane.
