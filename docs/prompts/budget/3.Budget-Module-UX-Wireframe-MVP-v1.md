# Budget Module — UX Wireframe (MVP v1)

**1\. Core UX principle**

Budget must feel like:

Select budget → see allocations → adjust allocations

Not:

Open long financial form → scroll → guess

**2\. Budget landing page**

Use the same master-detail pattern as Strategy for consistency.

**2.1 Page header**

At the top:

- **Budget Management**
- subtitle: Create and manage budget allocations aligned to strategic plans.
- primary action: **New Budget**

**Layout**

- title and subtitle on left
- New Budget button on right

**2.2 Main body layout**

Two columns:

**Left column — Budget list**

Shows all budgets

Each row/card shows:

- budget_name
- fiscal_year
- status badge

Example:

FY2026 Budget

2026 • Draft

Selected row must be clearly highlighted.

**Right column — Selected budget overview**

Shows the selected budget

Sections:

**A. Overview**

- Budget Name
- Fiscal Year
- Status badge
- Linked Strategic Plan
- Currency

**B. Financial summary**

- Total Budget Amount
- Allocated Amount
- Remaining Amount

**C. Actions**

- Open Budget Builder
- Edit Budget

**D. Allocation summary**

- Program count
- Allocated programs count
- Unallocated programs count

**2.3 Empty state**

If no budgets exist:

No budgets yet. Create one to begin.

And show only:

- page header
- create button
- empty-state panel

**2.4 Populated state**

When budgets exist:

- empty-state disappears
- first budget auto-selected
- detail panel updates accordingly

**3\. Budget creation UX**

This must stay minimal.

**3.1 Fields shown**

Only:

- Budget Name
- Fiscal Year
- Strategic Plan
- Currency
- Total Budget Amount (optional in v1, but recommended)

**3.2 Guidance text**

At top of create screen:

Create a budget aligned to a strategic plan. You will allocate funds to programs in the next step.

**3.3 Primary action**

- **Save and Continue**

Do not make the user guess what happens next.

**3.4 Next-step hint**

Below button:

Next: Allocate budget amounts to strategic programs.

**4\. Budget Builder UX**

This is the core working screen.

**4.1 Goal**

The user should be able to:

- see the programs from the linked Strategic Plan
- select one
- assign or edit a budget allocation
- understand totals immediately

**4.2 Layout**

Use a two-pane builder, but simpler than Strategy.

**Top summary bar**

Always visible:

- Total Budget
- Allocated Amount
- Remaining Amount

Example:

Total Budget: KES 8,000,000

Allocated: KES 5,000,000

Remaining: KES 3,000,000

If total is not set:

Allocated: KES 5,000,000

**Left pane — Program allocation list**

This is not a deep tree yet. It is a structured list.

Each row shows:

- Program title
- Allocation amount (or “Not allocated”)
- selection highlight

Example:

Healthcare Access KES 5,000,000

Workforce Development KES 3,000,000

Optional visual cue:

- allocated = subtle green/neutral amount
- unallocated = muted “Not allocated”

**Right pane — Allocation editor**

When a Program is selected, show:

**Section 1 — Program**

- Program Title (read-only)
- Program Description (read-only, optional collapsed)

**Section 2 — Allocation**

- Allocation Amount
- Notes (optional)

**Section 3 — Summary**

- Current Budget Total
- Total Allocated Before This Change
- Remaining After Save (preview if easy)

**Action**

- Save Allocation

**4.3 Empty selection state**

If no program selected:

Select a program from the left to view or edit its allocation.

**4.4 Builder behavior rules**

**Rule 1**

Programs load from the selected Budget’s Strategic Plan only.

**Rule 2**

One allocation per Program per Budget.

**Rule 3**

Selecting a Program updates the right pane.

**Rule 4**

Saving creates or updates the allocation in place.

**Rule 5**

Do not navigate away for normal editing.

**Rule 6**

After save:

- row amount updates
- totals update
- selection remains stable

**7\. Budget UX rules (non-negotiable)**

**7.1 No long forms**

Budget creation and editing must not become long accounting forms.

**7.2 No account/cost-center leakage**

Do not expose ERPNext accounting concepts in this UX.

**7.3 One obvious next action**

Landing page → Open Builder  
Builder → Save Allocation

**7.4 Totals always visible**

User must not mentally compute totals.

**7.5 Allocation list must be scannable**

Programs and amounts must be readable at a glance.
