Budget Module Exit Checklist

Here is the **Budget module exit checklist**. Use it as the hard gate before Procurement.

**As of 2026-04-21:** Revised to include **Rejected** and minimal rejection/resubmit (see [8.a.Budget-Approval-Flow - 2.md](./8.a.Budget-Approval-Flow%20-%202.md)). Literal status stays **Rejected** until re-submit; there is no “return to Draft” status.

**Budget MVP Exit Checklist**

**A. Core data model**

-   Budget exists with required fields
-   Budget Allocation exists with required fields
-   Budget links to one Strategic Plan
-   Allocation links to one Program
-   (budget, program) uniqueness enforced
-   allocation amount must be > 0
-   cross-plan allocation blocked

**B. Landing page**

-   Budget landing route loads
-   empty state works
-   populated state works
-   first budget auto-selects
-   selected budget panel updates correctly
-   New Budget CTA visible in correct place
-   Open Budget Builder is primary action (where role allows)
-   KPI values render correctly without truncation

**C. Budget create flow**

-   minimal create form implemented
-   required fields enforced
-   Save and Continue works
-   lands in Budget Builder for Draft budget

**D. Budget Builder**

-   builder route loads
-   linked Programs load from selected Strategic Plan
-   left list shows allocated vs unallocated programs
-   selecting a Program updates editor
-   allocation save creates/updates correctly
-   totals update immediately
-   back navigation to Budget landing is clear

**E. Governance / approval flow**

-   statuses implemented: **Draft / Submitted / Approved / Rejected**
-   valid transitions enforced server-side (including **Submitted → Rejected**, **Rejected → Submitted**; **Approved** terminal; no **Submitted → Draft**)
-   Strategy Manager can submit (from Draft or Rejected)
-   Planning Authority can approve (Submitted → Approved) and **reject** (Submitted → Rejected) with required reason
-   Strategy Manager cannot approve (same as before)
-   invalid transitions blocked server-side
-   Submitted budget becomes read-only (content + allocations)
-   Approved budget becomes read-only
-   **Rejected** budget is editable again (content + allocations) for revision before re-submit
-   Budget Allocation edits blocked in Submitted/Approved; allowed in Draft/Rejected
-   landing page actions change by status and role (incl. reject modal, re-submit)
-   builder shows read-only state clearly for Submitted/Approved; editable for Draft/Rejected

**F. Roles / permissions**

-   Strategy Manager sees Budget workspace
-   Planning Authority sees Budget workspace
-   Requisitioner does not see Budget workspace
-   backend permission enforcement exists
-   route protection exists
-   UI visibility matches backend permissions

**G. Seed data**

-   seed\_budget\_empty works
-   seed\_budget\_basic works
-   seed\_budget\_extended works
-   seed is idempotent
-   seeded budgets use exact expected names
-   seeded allocations match exact expected values
-   approval-state coverage exists in seed or test setup (Submitted/Approved scenarios; **Rejected** may come from tests or manual reject)

**H. Smoke tests**

-   landing empty-state test passes
-   landing populated-state test passes
-   selection test passes
-   create-flow test passes
-   builder-load test passes
-   allocation-save test passes
-   totals test passes
-   role visibility tests pass
-   approval-flow tests pass
-   **rejection-flow tests pass** (Python: `test_budget_rejection.py`, extended B51/B53; Playwright: `budget-rejection.spec.ts` where credentials allow)

**I. Downstream trust boundary**

-   **Approved** budgets are the only status stable enough for Procurement to treat as committed baseline
-   Procurement will not rely on **Draft**, **Submitted**, or **Rejected** budgets by mistake (only **Approved**)
-   governance gaps and residual risks are documented ([Budget-B5.8-Governance-Review.md](./Budget-B5.8-Governance-Review.md), tracker)

**Rule before moving on**

You do **not** move to Procurement unless:

All sections **E, F, H, and I** are green.

Because that is where the budget becomes trustworthy.
