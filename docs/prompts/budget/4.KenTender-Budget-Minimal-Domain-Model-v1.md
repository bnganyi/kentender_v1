**KenTender — Budget Minimal Domain Model v1**

**Strict Implementation Specification**

**1\. Scope**

This specification defines the **minimum Budget data model** required for v1.

It supports:

- Budget creation
- Budget listing in workspace
- selection of a linked Strategic Plan
- Program-level budget allocation
- total / allocated / remaining display
- future downstream integration with Procurement

It does **not** support yet:

- Objective-level allocation
- budget approval workflow
- budget revisions
- commitment accounting
- ERPNext GL integration
- variance tracking

**2\. Design decisions (locked)**

1.  Use **typed entities**, not generic financial rows:
    - Budget
    - Budget Allocation
2.  Budget is linked to exactly **one Strategic Plan**
3.  Allocation is done at **Program level only** in v1
4.  One Program can have **at most one allocation per Budget**
5.  Budget is a **planning object**, not an accounting object

**3\. Entity relationship model**

Strategic Plan

└── Budget

└── Budget Allocation → Strategy Program

Constraint:

Budget Allocation.program.strategic_plan == Budget.strategic_plan

**Autoname**

**Preferred**

**Example**

**Allocation-level constraints**

| **Constraint ID** | **Rule** |
| --- | --- |
| ALLOC-001 | budget required |
| ALLOC-002 | program required |
| ALLOC-003 | amount > 0 |
| ALLOC-004 | program.strategic_plan == budget.strategic_plan |
| ALLOC-005 | procuring_entity == budget.procuring_entity |
| ALLOC-006 | strategic_plan == budget.strategic_plan |

| ALLOC-008 | if budget.total_budget_amount exists, then sum(all allocations for budget) <= budget.total_budget_amount |

**5\. Computed values (not stored unless needed for performance)**

These values should be treated as **derived values**, not primary stored fields.

**5.1 Budget.allocated_amount**

sum(Budget Allocation.amount where allocation.budget = this budget)

**5.2 Budget.remaining_amount**

if total_budget_amount is not null:

total_budget_amount - allocated_amount

else:

null

**5.3 Budget.allocated_program_count**

count(Budget Allocation where allocation.budget = this budget)

**5.4 Budget.program_count**

count(Strategy Program where program.strategic_plan = budget.strategic_plan)

**5.5 Budget.unallocated_program_count**

program_count - allocated_program_count

**6\. UI-binding rules**

These are part of the spec because the model must support the builder without ambiguity.

**6.1 Budget create screen fields**

Only show:

- Budget Name
- Procuring Entity
- Fiscal Year
- Strategic Plan
- Currency
- Total Budget Amount
- Notes

Do not show:

- versioning fields
- supersedes fields
- internal IDs
- derived counts

**6.2 Budget landing page selected panel**

Show:

- Budget Name
- Fiscal Year
- Status
- Linked Strategic Plan
- Currency
- Total Budget Amount
- Allocated Amount
- Remaining Amount
- Program count
- Allocated program count
- Unallocated program count

**6.3 Budget builder editor**

When a Program is selected, show:

- Program Title (read-only)
- Allocation Amount
- Notes
- Save Allocation

Do not show:

- objective fields
- accounting fields
- account/cost center fields
- line-item breakdown

**8\. Permission-binding implications**

This is not the full roles spec, but the model must support it.

**Budget**

- entity-scoped visibility
- editable only by Budget-owning role in Draft state

**Budget Allocation**

- inherits visibility from Budget
- inherits mutability from Budget

Therefore:

- procuring_entity and strategic_plan should be present or derivable cleanly

**9\. Validation implementation rules**

Implement validation in backend, not only UI.

**Budget validation methods should cover:**

- required fields
- entity-plan consistency
- non-negative total budget
- uniqueness/version rule

**Budget Allocation validation methods should cover:**

- budget-program plan consistency
- positive amount
- unique program per budget
- total allocation ceiling if total budget is defined

**12\. Authoritative business rules summary**

| **Rule ID** | **Rule** |
| --- | --- |
| BUD-001 | Budget must belong to one Procuring Entity |
| BUD-002 | Budget must link to one Strategic Plan |
| BUD-003 | Strategic Plan must belong to same Procuring Entity as Budget |
| BUD-004 | Allocation is only allowed to Programs from the Budget’s Strategic Plan |
| BUD-005 | One Budget Allocation per Program per Budget |
| BUD-006 | Allocation amount must be positive |
| BUD-007 | Total allocations must not exceed total budget if total budget is set |
| BUD-008 | Budget is planning-layer only, no accounting fields in v1 |
