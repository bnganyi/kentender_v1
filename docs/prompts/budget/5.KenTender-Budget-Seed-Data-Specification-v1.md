# KenTender — Budget Seed Data Specification v1

**1\. Purpose**

This seed set enables:

- Budget workspace testing (empty + populated)
- Budget creation validation
- Budget builder functionality
- allocation behavior
- totals computation
- role-based UI testing
- future Procurement alignment

It must be:

deterministic

minimal

realistic

repeatable

**2\. Dependencies**

Budget seed depends on:

**Required seed packs**

seed_core_minimal

seed_strategy_basic

- Budget must link to Strategic Plan
- Allocations must reference Strategy Programs

**7\. Negative test scenarios (important)**

These should be seeded or tested via UI:

**7.1 Over-allocation (blocked)**

Attempt:

5000000 + 4000000 > 8000000

Expected:

- validation error
- no save

**7.2 Duplicate allocation (blocked)**

Attempt:

- create second allocation for same program

Expected:

- validation error
- no duplicate record

**7.3 Cross-plan allocation (blocked)**

Attempt:

- allocate program from different Strategic Plan

Expected:

- validation error

**10\. Seed implementation rules**

**10.1 Idempotency**

Each seed must:

- create if missing
- update if exists
- never duplicate

**10.2 Order of execution**

seed_core_minimal

→ seed_strategy_basic

→ seed_budget_basic / seed_budget_extended

**10.3 Deterministic naming**

Always use exact names:

- FY2026 Budget
- FY2027 Budget

Never use:

- Budget 1
- Test Budget

**10.4 No randomness**

Amounts must always be:

- 5000000
- 3000000
- 6000000
- etc.

So tests remain stable.
