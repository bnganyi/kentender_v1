## 1. Implementation Mandate

Cursor must implement the **STD Engine — Works Configuration** as a structured legal configuration engine, not a file upload feature.

The implementation must ensure that DOC 1 Works Building STD is represented as structured, versioned, source-traced seed data and that tender-specific STD instances generate all required downstream outputs.

The following are mandatory:

1. STD template family, version, profile, section, clause, parameter, form, BOQ, mapping, readiness, output, addendum impact, and audit records must exist as first-class structured data.
2. DOC 1 must be loaded through maintainable seed files, not hard-coded UI logic.
3. ITT and GCC must be locked.
4. TDS and SCC must be parameter-only.
5. Section III must generate DEM and must not be manually extended downstream.
6. Section IV must generate DSM.
7. Section V BOQ must be structured.
8. Supplier BOQ quantity editing must be impossible.
9. Arithmetic correction must exist only in DEM.
10. Contract carry-forward must exist in DCM.
11. Published outputs must be immutable and superseded by new versions only.
12. Post-publication changes must use addendum impact analysis and regeneration.
13. Tender Management v2 must consume STD outputs and must not redefine STD-owned rules.
14. UI must be a workbench and guided configuration surface, not raw DocType navigation.
15. Smoke Contract tests must pass with zero critical failures.

## 2. Non-Negotiable Anti-Patterns

Cursor must not implement any of the following:

| Anti-Pattern | Reason |
|---|---|
| Uploading STD PDF as authoritative tender rules | Breaks structured source-of-truth architecture. |
| Treating BOQ spreadsheet as authoritative without structured import | Breaks Works BOQ control and arithmetic correction rules. |
| Editing generated DSM/DOM/DEM/DCM directly | Breaks derived model integrity. |
| Letting Tender Management create manual submission requirements | Duplicates and conflicts with DSM. |
| Letting Tender Management create manual evaluation criteria | Violates disclosed criteria/source-of-truth principle. |
| Letting Bid Opening add opening fields outside DOM | Breaks opening traceability. |
| Letting Evaluation add rules outside DEM | Creates hidden evaluation criteria. |
| Letting Contract Management override DCM source rules | Breaks tender-contract binding. |
| Editing active template versions in place | Breaks legal versioning. |
| Editing published STD instances directly | Breaks immutability. |
| Overwriting published generated outputs | Breaks audit trail. |
| UI-only authorization | Server-side enforcement is mandatory. |
| Building UI only as raw list/detail DocTypes | Users need guided workbench workflows. |
| Collapsing all seed data into one monolithic fixture | Prevents maintenance, review, and supersession. |
