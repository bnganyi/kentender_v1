## 2. Purpose

The purpose of this document is to define the smoke test for the Procurement Officer Tender Configuration POC.

The smoke test shall verify that a procurement officer can create and configure a draft Works tender from a controlled STD template package using a guided workflow, without editing package internals or raw JSON, and can proceed through validation, required forms generation, representative BoQ handling, preview generation, and audit review.

This smoke test is not a production certification test. It verifies that the officer-facing POC is fit to support the next project decision.

---

## 3. Smoke Test Principle

The smoke test shall implement the principle:

> The officer POC passes only if the procurement officer can safely configure a draft tender through guided UI and the system can validate, explain, generate, preview, and audit the result without exposing template internals or drifting into downstream procurement workflows.

The test shall confirm:

1. officer can start a tender from an allowed STD template;
2. officer sees template summary, not package internals;
3. guided configuration groups are visible and usable;
4. conditional fields respond to major Works scenarios;
5. configuration syncs to canonical engine input;
6. validation feedback is officer-friendly;
7. required forms checklist is generated and understandable;
8. representative BoQ status is visible and bounded;
9. preview generation is blocked when unsafe and succeeds when valid;
10. audit metadata is visible;
11. out-of-scope actions are absent.

---

## 4. Scope of the Smoke Test

This smoke test covers:

1. officer tender creation and template selection;
2. guided configuration groups;
3. conditional field behavior;
4. configuration sync behavior;
5. validation feedback;
6. required forms checklist;
7. BoQ status and representative rows;
8. tender-pack preview generation;
9. preview blocking for invalid configurations;
10. audit summary;
11. POC state indicators;
12. permission and safety spot checks;
13. scope boundary checks.

---

## 5. Non-Scope

The smoke test shall not test or require:

1. tender publication;
2. final PDF generation;
3. bidder portal;
4. bid submission;
5. bid opening;
6. evaluation;
7. award;
8. contract execution;
9. production BoQ authoring;
10. Excel BoQ import/export;
11. bidder pricing;
12. external integrations;
13. full production approval workflow;
14. legal certification of preview output.

If any of these appear during smoke testing, they shall be recorded as scope drift.

## 9. Smoke Path Overview

The smoke path shall be executed in this order:

1. Open officer tender list/workspace.
2. Start new tender.
3. Select STD-WORKS-POC template.
4. Confirm template summary and POC boundary notice.
5. Complete primary sample-equivalent guided configuration.
6. Confirm conditional field behavior.
7. Save/sync configuration.
8. Validate configuration.
9. Generate required forms checklist.
10. Generate/check representative BoQ.
11. Generate tender-pack preview.
12. Review preview and audit summary.
13. Test positive conditional branches.
14. Test negative validation branches.
15. Test BoQ readiness branch.
16. Confirm permission and safety boundaries.
17. Confirm out-of-scope actions are absent.
18. Capture evidence and log issues.

## 31. Pass / Fail Criteria

### 31.1 Pass Criteria

The Officer POC smoke test passes if:

1. officer can start a tender from the allowed STD template;
2. guided configuration groups are visible and usable;
3. raw package internals are not exposed for officer editing;
4. conditional field behavior works for major branches;
5. configuration syncs and validates;
6. valid primary sample-equivalent configuration passes validation;
7. required forms checklist generates 15 forms and is understandable;
8. representative BoQ shows 9 rows and expected category coverage;
9. preview generates for valid configuration;
10. preview is blocked for invalid configurations;
11. audit metadata includes package and configuration hashes;
12. stale output behavior is visible or conservatively handled;
13. out-of-scope downstream actions are absent;
14. required evidence is captured.

### 31.2 Fail Criteria

The smoke test fails if any of the following occurs:

1. officer cannot create/init tender from template;
2. guided configuration is not usable;
3. officer must edit raw JSON to complete workflow;
4. package internals are editable by officer;
5. validation cannot run or is not understandable;
6. preview can be generated as valid despite blockers;
7. forms checklist is missing or not explainable;
8. BoQ status is opaque or missing;
9. audit hashes are missing without warning;
10. publication/PDF/bidder/evaluation/award/contract scope appears;
11. major permission boundaries fail.

## 35. Exit Condition

Officer POC Step 9 is complete when the smoke test has been executed, the result table is completed, required evidence has been captured, issues have been logged and dispositioned, and the Procurement Officer Tender Configuration POC has either passed or failed with clear reasons.
