# STD Administration Console POC — implementation tracker

**Purpose:** Single place to record admin-console step status, evidence (tests, Playwright), acceptance, and exit-condition fulfilment for the **STD Administration Console POC** workstream (read-only package visibility, validation, rule trace, demo workspace, inspectors, preview/audit, smoke, go/no-go).

**Parent workstream:** [STD-WORKS-POC tracker](../IMPLEMENTATION_TRACKER.md) — package, engine, and tender POC actions. **Issues log (shared):** [`../ISSUES_LOG.md`](../ISSUES_LOG.md) — use ids **`STD-ADMIN-NNN`** for this workstream (and **`STD-POC-NNN`** for parent POC).

**Depends on:** STD-WORKS-POC through Step **16** minimum (engine + tender actions + preview); Step **17** evidence optional for reviewer context.

**Agent rule:** [`.cursor/rules/kentender-std-admin-console-implementation.mdc`](../../../../../.cursor/rules/kentender-std-admin-console-implementation.mdc)

## Rules of engagement

1. **Scope document is law for boundaries** — Read [`1. std_admin_console_scope_document.md`](1.%20std_admin_console_scope_document.md) before coding; honour **Do / Do not** boundary prompts in each admin spec.
2. **No template authoring** — Package JSON and rules remain read-only in UI; no procurement officer wizard, bidder portal, PDF, or production BoQ in this workstream.
3. **TDD; Playwright for UI** — Any new Desk-visible admin UI requires automated tests + **Playwright** where the spec defines UI smoke; cite commands in **Evidence**.
4. **Reuse engine and loader** — Call `std_template_loader` / `std_template_engine` and existing `Procurement Tender` whitelisted methods; do not duplicate rule logic in client scripts.
5. **Exit conditions** — Mark **`Done`** only when that step’s spec acceptance/exit items are met **and** evidenced. Use **`Partial`** / **`Blocked`** as in the parent tracker discipline.

### Per-step completion (before Done)

- Admin scope + IA read for the tranche  
- Step spec: rationale, inputs, outputs reviewed  
- Boundaries + acceptance criteria satisfied (reference in **Notes**)  
- TDD + Playwright evidence in **Evidence**
