Supplier Management — UI Specification v1.0

*(Workbench + Builder + External View, aligned with your refined UX principles)*

This follows your system-wide standards:

-   queue-first interaction model
-   compact header → KPI strip → lightweight controls → dominant work area
-   no clutter, no redundant controls
-   governance-driven UI (not just forms)

Under Procurement workspace:

**7\. Actions model (critical)**

**Actions must be:**

-   visible only when valid
-   located top-right in detail panel
-   never duplicated in multiple places

**Example: Under Review state**

\[Approve\] \[Return\] \[Reject\]

NOT:

-   Submit
-   Suspend
-   Blacklist

**8\. Filters (minimal)**

Filters button only (no heavy bar):

-   Category
-   Compliance status
-   Risk level

Show active filters inline:

Filters: Category = ICT ×

**9\. Empty states**

Example:

No suppliers in “Submitted”

\[View All\] \[Create Supplier\]

**10\. Visual hierarchy enforcement**

Priority must be:

1.  Queue selection
2.  Supplier list + detail
3.  KPI strip
4.  Tabs
5.  Filters

If control area dominates → incorrect

**11\. Acceptance criteria**

UI is correct only if:

-   queue-first interaction is clear
-   no multi-row control clutter
-   statuses are separated (approval/operational/compliance)
-   actions are role/state-driven
-   supplier detail is the main focus
-   builder is guided, not a long form
-   external model is respected (even if no UI yet)
-   no raw IDs shown
-   filters are minimal and hidden by default
