**Supplier constraints**

| **Rule** | **Constraint** |
| --- | --- |
| SUP-001 | supplier\_code must be unique |
| SUP-002 | primary\_contact\_email must be unique |
| SUP-003 | Company/NGO/Consortium requires registration\_number |
| SUP-004 | External supplier can edit only own Draft/Returned record |
| SUP-005 | Approved supplier identity fields require controlled internal update |
| SUP-006 | Blacklisted supplier cannot be edited or reactivated casually |
| SUP-007 | Operational status determines downstream eligibility |

**Supplier Creation flow update**

For reusability supplier registration should create or link:

ERPNext Supplier  
→ KenTender Supplier Governance Profile  
→ Supplier Category Assignments  
→ Supplier Documents  
→ API Access Grant

The governed profile is the record used by procurement workflows.

**Cross-entity invariants**

| **Invariant** | **Rule** |
| --- | --- |
| Eligibility | Supplier eligible only when Approved + Active + compliant + qualified for category |
| Documents | Required current documents must be Verified and unexpired |
| Categories | At least one Qualified category needed for category-specific eligibility |
| External access | External supplier can only access own records |
| Audit | Every lifecycle change creates Supplier Status History |
| Display | UI/API must use supplier code/name, not raw internal IDs |
| Downstream | Tendering must call eligibility service, not infer eligibility directly |
