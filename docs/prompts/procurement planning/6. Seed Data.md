Procurement Planning — Seed Data Specification (v1.0, Locked)

**1\. Purpose**

This seed dataset must:

-   provide **complete coverage** of Procurement Planning flows
-   support **template-driven package creation**
-   enable **end-to-end flow testing**:

DIA → Procurement Planning → Package → Ready for Tender

-   include **normal, high-risk, and emergency cases**
-   ensure **traceability to Budget and DIA**

**2\. Seed Structure (Strict Order)**

Seed must be created in this order:

1\. Profiles

2\. Templates

3\. Procurement Plan

4\. DIA References (assumed existing)

5\. Procurement Packages

6\. Package Lines
