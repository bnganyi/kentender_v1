**Purpose**

**Critical Constraints**

-   demand\_id must exist and be **Approved (or Planning Ready)**
-   A demand may appear:
    -   once (strict model) OR
    -   partially across multiple packages (if you allow split)
-   For v1, prefer:

one demand → one package (simpler, safer)

-   SUM(package\_lines.amount) = package.estimated\_value

**10\. Global Constraints (Non-negotiable)**

**10.1 No orphan package**

package.template\_id must exist

**10.2 No orphan line**

package\_line.demand\_id must exist

**10.3 No mutation of upstream modules**

-   Planning cannot modify:
    -   DIA
    -   Budget

**10.4 Approval locks**

-   Approved Plan → locked
-   Approved Package → locked

**10.5 Traceability must be complete**

You must always be able to traverse:

Package → Package Line → DIA → Budget Line

**12\. Minimal v1 Simplifications (Intentional)**

To avoid over-complication:

-   allow **one demand per package** initially (optional)
-   do not implement:
    -   complex grouping engine yet
    -   partial demand splitting (can come later)
-   keep templates simple JSON configs
