Procurement Planning — Smoke Contract (v1.0, Locked)

**1\. Purpose**

This smoke contract proves that Procurement Planning works as a governed planning layer, not just a package list.

It must verify that:

-   approved DIA demand can be ingested safely
-   template application creates coherent packages
-   package lines preserve traceability
-   method and profile defaults are applied correctly
-   approvals and locking behave correctly
-   Ready for Tender handoff is trustworthy
-   role permissions hold under real flows

**2\. Scope**

**Included**

-   Procurement Plan
-   Procurement Package
-   Package Line
-   Procurement Template
-   Risk/KPI/Decision/Vendor profiles
-   template application
-   package workflow
-   plan workflow
-   package approval
-   Ready for Tender handoff
-   role-based visibility and actions

**Excluded**

-   Tendering
-   supplier onboarding/management logic
-   contract execution
-   stores/assets
-   payment/accounting

**3\. Required dependencies**

This smoke suite assumes the following upstream pieces already work:

-   Strategy module
-   Budget module with:
    -   Budget Line
    -   Budget Reservation
    -   budget control services
-   DIA module with:
    -   approved demand records
    -   budget line references
    -   reservation references where applicable

**Planning Authority**

-   Submitted / Pending Approval
-   High-Risk / Override cases if present

-   casual planner edit controls

**Auditor**

Should see:

-   package detail and traceability

Should not see:

-   any mutating actions
